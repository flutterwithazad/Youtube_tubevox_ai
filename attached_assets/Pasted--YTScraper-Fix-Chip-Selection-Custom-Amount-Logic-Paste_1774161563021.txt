## PROBLEMS TO FIX

There are 3 bugs in the current chip + custom amount logic:

### Bug 1 — 5,000 and 10,000 chips are disabled when they shouldn't be
Currently: chips are disabled if balance < chip value. This is WRONG.
A user with 8 credits SHOULD be able to click the 5,000 chip — they just
won't be able to afford it, and that's shown as a warning, not blocked.

Fix: ALL chips (All, 5000, 10000, Custom) are ALWAYS clickable. Never disabled.
The only time anything is blocked is the "Scrape Now" button itself.

### Bug 2 — Custom input caps the value at balance and auto-replaces
Currently: when user types 100 with balance=8, it auto-replaces to 8.
This is WRONG. The user should be able to type any number freely.
If they type more than their balance, show a warning — but do NOT change
what they typed. They can see the shortfall and decide to buy more credits.

### Bug 3 — No "shortfall" message when selected amount > balance
Currently: there's no helpful message telling the user how many more credits
they need. Add this for chips AND custom input.
