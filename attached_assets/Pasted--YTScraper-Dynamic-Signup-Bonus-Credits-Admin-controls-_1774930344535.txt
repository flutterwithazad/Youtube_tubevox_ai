## WHAT WE ARE BUILDING

A single source of truth for the signup bonus credit amount:
- Admin sets it in the admin panel → saves to `platform_settings` table
- New users receive that exact amount on signup (via DB trigger)
- Landing page, FAQ, pricing section, dashboard — all read it live from DB
- If admin changes it from 500 to 1000 → everywhere updates automatically
- No hardcoded numbers anywhere in the codebase

---

## THE DATA SOURCE

The value already exists in `platform_settings` table:
```
key   = 'free_plan_credits'
value = '500'   ← admin changes this number
```

This is the ONLY place the number lives.
Every part of the app reads from here. Nothing is hardcoded.

## PART 1 — DATABASE: Ensure trigger uses dynamic value

The `handle_new_user()` trigger runs when a user signs up.
It must read `free_plan_credits` from `platform_settings` dynamically,
NOT use a hardcoded number.

Run this in Supabase SQL Editor to update the trigger:

```sql
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
DECLARE
  v_plan_id      UUID;
  v_sub_id       UUID;
  v_bonus_credits INT;
BEGIN
  -- ── 1. Get free plan ────────────────────────────────────────
  SELECT id INTO v_plan_id
  FROM public.plans
  WHERE price_monthly = 0
    AND is_active = true
  ORDER BY created_at ASC
  LIMIT 1;

  -- ── 2. Insert profile ────────────────────────────────────────
  INSERT INTO public.profiles (id, email, full_name)
  VALUES (
    NEW.id,
    NEW.email,
    COALESCE(NEW.raw_user_meta_data->>'full_name', '')
  );

  -- ── 3. Create subscription on free plan ─────────────────────
  IF v_plan_id IS NOT NULL THEN
    INSERT INTO public.subscriptions (user_id, plan_id, status, billing_cycle)
    VALUES (NEW.id, v_plan_id, 'active', 'monthly')
    RETURNING id INTO v_sub_id;

    UPDATE public.profiles
    SET current_plan_id        = v_plan_id,
        current_subscription_id = v_sub_id
    WHERE id = NEW.id;
  END IF;

  -- ── 4. Read bonus amount DYNAMICALLY from platform_settings ──
  -- NEVER hardcode this number
  SELECT COALESCE(value::INT, 500)
  INTO v_bonus_credits
  FROM public.platform_settings
  WHERE key = 'free_plan_credits';

  -- Fallback to 500 if setting is missing or not a valid integer
  IF v_bonus_credits IS NULL OR v_bonus_credits <= 0 THEN
    v_bonus_credits := 500;
  END IF;

  -- ── 5. Grant signup bonus credits ───────────────────────────
  INSERT INTO public.credit_ledger (
    user_id,
    amount,
    source_type,
    description,
    balance_after
  ) VALUES (
    NEW.id,
    v_bonus_credits,
    'signup_bonus',
    'Welcome bonus — ' || v_bonus_credits || ' free credits on signup',
    v_bonus_credits
  );

  RETURN NEW;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;
```

Verify the trigger is attached:
```sql

FROM information_schema.triggers

-- Should return 1 row
```

## PART 2 — SHARED HOOK: usePlatformSetting

Create one shared hook that any component can use to read any platform setting.
This is the foundation — everything else uses this.

```typescript
// lib/hooks/usePlatformSetting.ts
'use client'
import { useEffect, useState, useCallback } from 'react'
import { createClient } from '@/lib/supabase/client'

export function usePlatformSetting(key: string): {
  value: string | null
  loading: boolean
  refetch: () => Promise<void>
} {
  const [value, setValue] = useState<string | null>(null)
  const [loading, setLoading] = useState(true)
  const supabase = createClient()

  const fetch = useCallback(async () => {
    const { data } = await supabase
      .from('platform_settings')
      .select('value')
      .eq('key', key)
      .single()
    setValue(data?.value ?? null)
    setLoading(false)
  }, [key])

  useEffect(() => { fetch() }, [fetch])

  return { value, loading, refetch: fetch }
}
```

Also create a server-side helper for Server Components and API routes:

```typescript
// lib/server/getPlatformSetting.ts
import { createClient } from '@/lib/supabase/server'

export async function getPlatformSetting(key: string): Promise<string | null> {
  const supabase = createClient()
  const { data } = await supabase
    .from('platform_settings')
    .select('value')
    .eq('key', key)
    .single()
  return data?.value ?? null
}
```

### Save handler in admin settings:

```typescript
const saveSetting = async (key: string, value: string) => {
  // Validate: must be a positive integer for free_plan_credits
  if (key === 'free_plan_credits') {
    const n = parseInt(value)
    if (isNaN(n) || n < 0) {
      toast.error('Signup bonus must be a positive number')
      return
    }
    if (n > 100000) {
      toast.error('Signup bonus cannot exceed 100,000 credits')
      return
    }
  }

  // Save to DB
  const response = await fetch(`/api/admin/settings/${key}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ value: String(value) }),
  })

  if (!response.ok) {
    toast.error('Failed to save setting')
    return
  }

  toast.success(
    key === 'free_plan_credits'
      ? `✓ Signup bonus updated to ${Number(value).toLocaleString()} credits. New signups will now receive this amount.`
      : '✓ Setting saved'
  )

  // Refresh the settings list
  await fetchSettings()
}
```
