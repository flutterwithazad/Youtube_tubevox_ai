## PROJECT OVERVIEW

Build a complete **Admin Panel** for the YouTube Comment Scraper SaaS.

The user dashboard already exists at `https://example.com/dashboard` (separate project).
The admin panel lives at `https://example.com/admin` and all sub-routes under `/admin/*`.

This is a **separate Next.js project** — do NOT mix with the user dashboard.
The root URL `/` should redirect to `/admin/login`.

The admin has **full control** over: users, plans, credits, jobs, payments,
YouTube API keys, platform settings, announcements, IP blocklist, and audit logs.

## ADMIN AUTHENTICATION SYSTEM

Admins use the `admin_users` table — completely separate from Supabase Auth.

### Login flow:
1. POST `/api/admin/auth/login` with `{ email, password }`
2. Look up `admin_users` by email
3. Verify password with `bcrypt.compare(password, admin.password_hash)`
4. Check `admin.is_active === true`
5. Load role from `admin_roles` WHERE `id = admin.role_id`
6. Sign JWT: `jwt.sign({ adminId, email, roleId, roleName, permissions: {...all can_* booleans} }, ADMIN_JWT_SECRET, { expiresIn: '8h' })`
7. Set httpOnly cookie `admin_token` with the JWT (sameSite: strict, secure: true)
8. Update `admin_users.last_login_at = NOW()`, `last_login_ip` from request
9. Log to `admin_audit_log`: action `admin.login`
10. Return `{ success: true }`

### Middleware protection:
Every `/admin/*` route except `/admin/login` must:
1. Read `admin_token` cookie
2. Verify with `jwt.verify(token, ADMIN_JWT_SECRET)`
3. If missing/invalid/expired: redirect to `/admin/login`
4. Attach decoded admin payload to request headers for use in server components

### Permission guard helper:
```typescript
// lib/check-permission.ts
export function requirePermission(
  adminPermissions: Record<string, boolean>,
  permission: keyof typeof adminPermissions
) {
  if (!adminPermissions[permission]) {
    throw new Error(`Forbidden: missing permission ${permission}`)
  }
}
```

### Create first super admin script:
Create `scripts/seed-admin.ts`:
- Hash password `Admin@123456` with bcrypt (salt rounds: 12)
- Insert into `admin_users` with role_id of super_admin
- Email: `admin@yourdomain.com`
- Print instructions to change password after first login

## AUDIT LOGGER UTILITY (use on EVERY write action)

```typescript
// lib/audit.ts
import { createSupabaseAdmin } from './supabase-admin'

export async function logAdminAction(params: {
  adminId: string
  action: string
  targetType?: string
  targetId?: string
  beforeValue?: object
  afterValue?: object
  ipAddress?: string
  userAgent?: string
  notes?: string
}) {
  const supabase = createSupabaseAdmin()
  await supabase.from('admin_audit_log').insert({
    admin_id: params.adminId,
    action: params.action,
    target_type: params.targetType ?? null,
    target_id: params.targetId ?? null,
    before_value: params.beforeValue ?? {},
    after_value: params.afterValue ?? {},
    ip_address: params.ipAddress ?? null,
    user_agent: params.userAgent ?? null,
    notes: params.notes ?? null,
  })
}
```

Actions to log (use these exact strings):
`admin.login`, `admin.logout`, `admin.create`, `admin.deactivate`, `admin.role_change`
`user.view`, `user.suspend`, `user.unsuspend`, `user.delete`, `user.plan_change`,
`user.credits_add`, `user.credits_deduct`, `user.force_logout`
`job.view`, `job.kill`, `job.retry`
`payment.refund`, `payment.view`
`api_key.add`, `api_key.pause`, `api_key.resume`, `api_key.reset`, `api_key.delete`
`settings.update`, `announcement.create`, `announcement.update`, `announcement.delete`, `announcement.toggle`
`ip.block`, `ip.unblock`
`plans.create`, `plans.update`, `package.create`, `package.update`, `package.delete`

Columns: Avatar + Name + Email | Role badge | 2FA badge | Active badge | Last login | Created by (admin name) | Date created | Actions

**Actions per row:**
- Edit role: slide-over to change role_id → UPDATE admin_users, log audit `admin.role_change`
- Deactivate: confirm → UPDATE is_active=false, deactivated_at=NOW(), deactivated_by=currentAdminId → log audit `admin.deactivate`
- Cannot deactivate your own account (check adminId === targetId)

**Create new admin** button → `/admin/admins/new`:
Form: full_name, email, password (shown once — admin must save it), role_id (select)
On submit → hash password with bcrypt → INSERT admin_users with created_by=currentAdminId → log audit `admin.create`

**Roles reference section (below table):**
Show the 3 roles (super_admin, support_admin, billing_admin) as a permissions grid — each role is a column, each permission is a row, checkmark or X showing what's allowed. Read-only reference.

## IMPLEMENTATION RULES

1. **All DB calls server-side only** — Route Handlers at `/api/admin/*`. Never call Supabase from browser components.

2. **Log every write action** via `logAdminAction()` utility. Every INSERT, UPDATE, DELETE. READ queries do NOT need logging.

3. **Confirm dialogs** before ALL destructive actions: suspend, delete, kill, refund, block, deactivate admin.

4. **Never expose sensitive fields:**
   - `youtube_api_keys.key_value` → show only first 8 chars + "••••••••"
   - `admin_users.password_hash` → never return from any API
   - `admin_users.totp_secret` → never return from any API
   - `user_sessions.session_token_hash` → never return from any API

5. **Server-side pagination everywhere** — use `.range(from, to)` with Supabase. 25 rows per page default.

6. **Real-time** — use Supabase Realtime channel subscription on `jobs` table filtered to `status=running` for the live running jobs count on the overview page.

7. **Error handling** — wrap every page in an error boundary. Show "Failed to load — please refresh" on query errors.

8. **Desktop only** — no need for mobile responsive. Show "Use a desktop browser" below 768px.

9. **Loading states** — every table shows `<SkeletonTable>` while loading. Stat cards show skeleton rectangles.

10. **TypeScript strict** — define `Database` types for all tables in `types/database.ts`.
